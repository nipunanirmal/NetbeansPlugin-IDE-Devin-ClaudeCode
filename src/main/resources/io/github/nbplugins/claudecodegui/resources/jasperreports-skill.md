# JasperReports Skill

Guidelines for working with JasperReports in the Medicare/SAD-Viva project.

## 1. Compile-once, Cache Strategy

- `.jrxml` files are compiled to `.jasper` on first run using `JasperCompileManager.compileReportToFile(jrxmlPath, jasperPath)`
- On subsequent runs, the `.jasper` is loaded directly via `JRLoader.loadObject(jasperFile)` ΓÇö no recompile
- **Staleness check**: recompile only if `.jasper` is missing or older than its `.jrxml` (`jasperFile.lastModified() < jrxmlFile.lastModified()`)

## 2. Template Resolution via Classpath

- `.jrxml` files live under `src/com/medicare/reports/` and are resolved at runtime using `getClass().getClassLoader().getResource(path)` ΓÇö **never hardcoded absolute paths**
- The sibling `.jasper` path is derived by simple string replace: `jrxmlPath.replace(".jrxml", ".jasper")`

## 3. Filling with a Live JDBC Connection

- Reports are filled using `JasperFillManager.fillReport(jasperReport, new HashMap<>(), conn)` where `conn` is the live `DatabaseConnection.getConnection()`
- The SQL query is embedded in the `.jrxml` itself, not passed from Java

## 4. PDF Export

- Uses `JasperExportManager.exportReportToPdfFile(jasperPrint, outputPath)` ΓÇö the high-level one-liner API
- PDF support requires `itext-2.1.7.jar` on the classpath (bundled in `lib/ext/`)

## 5. Excel Export via Reflection

- No POI/xlsx JARs are guaranteed present, so the `JRXlsxExporter` is invoked entirely via reflection:
  - `Class.forName(...)`
  - `getDeclaredConstructor().newInstance()`
  - `getMethod(...).invoke(...)`
- This lets the project compile without POI; a `ClassNotFoundException` at runtime is the graceful failure signal

## 6. Output Directory Creation

- Before exporting, `outFile.getParentFile().mkdirs()` is called to ensure the output directory exists (default: `reports/output/`)

## 7. Output Path Naming Convention

- Default filenames follow: `reports/output/<report_name>_<yyyyMMdd_HHmmss>.<ext>`
- Generated by `getDefaultOutputPath()` using `java.time.LocalDateTime` + `DateTimeFormatter`

## 8. .jrxml Band Structure

| Band | Purpose |
|------|---------|
| `<title>` | Report header (clinic name, print timestamp, subtitle) |
| `<columnHeader>` | Table column labels, styled with `mode="Opaque"` + `backcolor` for colored header rows |
| `<detail>` | One row per result-set record |
| `<pageFooter>` | Page number using built-in `$V{PAGE_NUMBER}` |
| `<summary>` | Totals row using built-in `$V{REPORT_COUNT}` |
| `<group>` | Used in `invoice_report.jrxml` to group detail rows by `invoice_code`, with `<groupHeader>` (sub-header per invoice) and `<groupFooter>` (per-invoice totals) |

## 9. Field & Expression Syntax in .jrxml

- DB columns declared as `<field name="..." class="java.lang.String|java.sql.Date|java.math.BigDecimal|..."/>`
- Referenced in expressions as `$F{field_name}`
- **String concatenation** inside expressions: `$F{first_name} + " " + $F{last_name}`
- **Date formatting** inline: `new java.text.SimpleDateFormat("dd/MM/yyyy").format($F{invoice_date})`
- `isBlankWhenNull="true"` on nullable fields (e.g. `blood_group`, invoice item fields) to suppress null output

## 10. Number/Date Formatting Patterns

- **Dates**: `pattern="dd/MM/yyyy"` or `"dd/MM/yyyy HH:mm"` directly on `<textField>`
- **Currency**: `pattern="# ##0.00"` on `BigDecimal` fields

## 11. Page Orientation & Sizing

- **Portrait**: `pageWidth="595" pageHeight="842"` (A4)
- **Landscape**: `pageWidth="842" pageHeight="595"` (A4 rotated) ΓÇö used for wide tables like the patient list

## 12. Opening the Generated File

- After export, `Desktop.getDesktop().open(new File(outputPath))` opens the file with the OS default viewer
- Guarded by `Desktop.isDesktopSupported()` before calling; falls back to a "saved to" dialog if `open()` throws

## Key Implementation Pattern

```java
// 1. Resolve template path via classpath
String jrxmlPath = "com/medicare/reports/patient_list.jrxml";
URL resource = getClass().getClassLoader().getResource(jrxmlPath);

// 2. Compile if needed (check staleness)
String jasperPath = jrxmlPath.replace(".jrxml", ".jasper");
File jasperFile = new File(jasperPath);
File jrxmlFile = new File(resource.toURI());

if (!jasperFile.exists() || jasperFile.lastModified() < jrxmlFile.lastModified()) {
    JasperCompileManager.compileReportToFile(jrxmlFile.getAbsolutePath(), jasperFile.getAbsolutePath());
}

// 3. Load compiled report
JasperReport jasperReport = (JasperReport) JRLoader.loadObject(jasperFile);

// 4. Fill with JDBC connection
Connection conn = DatabaseConnection.getConnection();
JasperPrint jasperPrint = JasperFillManager.fillReport(jasperReport, new HashMap<>(), conn);

// 5. Export to PDF
String outputPath = getDefaultOutputPath("patient_list", "pdf");
JasperExportManager.exportReportToPdfFile(jasperPrint, outputPath);

// 6. Open with default viewer
if (Desktop.isDesktopSupported()) {
    Desktop.getDesktop().open(new File(outputPath));
}
```
